A clean, modular, production-ready trading system for the Binance USDT-M Futures Testnet. It features a dual-interface architecture supporting both a robust Command-Line Interface (CLI) and an interactive Web User Interface (UI) dashboard. The application implements MARKET, LIMIT, and STOP-LIMIT orders using direct REST API integrations with manual HMAC-SHA256 signature calculation, client-side input validation, rotating logs, and structured exception handling.

## Project Overview

This project is a high-performance, modular trading utility designed for order routing and execution on the Binance USDT-M Futures exchange. It provides a flexible dual-interface setup:
- **CLI Mode**: Optimized for command-line execution, shell automation, and scripting.
- **Web UI Mode**: A lightweight, premium browser dashboard featuring real-time parameters preview, in-session key configuration, and scrolling live server logs.

By utilizing direct REST API endpoints and manual HMAC-SHA256 cryptographic request signing instead of third-party SDK packages, the system minimizes runtime dependency bloat, guarantees clean connection handling, and allows custom control over network timeout thresholds and error diagnostics.

---

## Features

- **Direct REST API Integration**: Fully manual signature generation (`HMAC-SHA256`) and connection management.
- **Order Types**: Support for **MARKET**, **LIMIT**, and **STOP_LIMIT** orders.
- **CLI Commands**: Rich argparse integration.
- **Strict Validation**: Local client-side checks for symbol casing/naming, quantity limits, and type-specific parameter requirements before sending API calls.
- **Robust Exception Handling**: Differentiates between validation, authentication, network timeouts, and raw API errors from Binance.
- **Detailed Logging**: Logs application lifecycle events, signed requests, raw JSON responses, and errors to `logs/trading.log` with a custom file rotator.

## Binance Testnet Configuration

### Create a Testnet Account
1. Open your browser and navigate to the [Binance Futures Testnet](https://testnet.binancefuture.com).
2. Log in using your email, phone number, or existing Binance credentials. (Note: You do not need to fund this account with real capital. It is loaded with mock USDT collateral upon registration).

### Generate API Keys
1. Once logged in, scroll down to the API Key section at the bottom of the screen.
2. Click **Generate API Key**.
3. Copy the **API Key** and **Secret Key** immediately. Keep them safe; the Secret Key will not be visible again.

### Setup Environment Variables (.env)
1. In the root directory of the project, copy the template `.env.example` into a new file named `.env`:
   ```bash
   cp .env.example .env
   ```
2. Open `.env` and input your keys:
   ```env
   BINANCE_API_KEY=your_copied_api_key
   BINANCE_API_SECRET=your_copied_secret_key
   ```

> [!WARNING]
> Never commit your `.env` file to version control. The `.gitignore` has been pre-configured to ignore `.env` files to prevent security leaks.

## How to Run & Examples

The CLI takes several parameters:
*   `--symbol`: The asset ticker symbol (must end in `USDT` e.g., `BTCUSDT`).
*   `--side`: The action `BUY` or `SELL`.
*   `--type`: The order constraint `MARKET`, `LIMIT`, or `STOP_LIMIT`.
*   `--quantity`: The amount of contract units to buy/sell (must be greater than 0).
*   `--price`: The limit price (required only for `LIMIT` and `STOP_LIMIT` orders).
*   `--stop-price`: The trigger price (required only for `STOP_LIMIT` orders).

### Example 1: Placing a MARKET BUY Order
Buy 0.001 BTC on the market:
```bash
python cli.py --symbol BTCUSDT --side BUY --type MARKET --quantity 0.001
```

### Example 2: Placing a LIMIT SELL Order
Set a limit order to sell 0.001 BTC at $65,000:
```bash
python cli.py --symbol BTCUSDT --side SELL --type LIMIT --quantity 0.001 --price 65000
```

### Example 3: Placing a STOP_LIMIT BUY Order (Bonus Feature)
Set a stop-limit order to buy 0.001 BTC with trigger price at $60,000 and limit execution price at $60,100:
```bash
python cli.py --symbol BTCUSDT --side BUY --type STOP_LIMIT --quantity 0.001 --price 60100 --stop-price 60000
```

### Running the Web UI Dashboard (Light Theme Only)
If you prefer a premium visual interface, you can launch the Web UI:
```bash
python -m uvicorn app:app --host 127.0.0.1 --port 8000
```
Then, open your web browser and navigate to:
```text
http://127.0.0.1:8000
```
From the web dashboard, you can:
- Switch tabs to toggle between BUY/SELL sides and MARKET/LIMIT/STOP_LIMIT order types.
- Input order parameters with immediate reactive preview.
- Toggle **Mock Mode** using the visual slider.
- Manage API credentials in-session using the Settings modal (keys are held in-memory and not stored on disk).
- Read the real-time scrolling console printing lines directly from `logs/trading.log`.

## Error Handling & Troubleshooting

The bot intercepts errors and prints friendly feedback. Common failures include:

1. **Validation Failure (Invalid Arguments)**:
   - *Cause*: Negative quantity, missing limit price for LIMIT, or lowercase symbols.
   - *Output*:
     ```text
     ==========================
     INPUT VALIDATION ERROR
     ==========================
     Message: Price is required for this order type.
     ```
2. **Missing Keys (AuthenticationError)**:
   - *Cause*: `.env` missing or configured with placeholders.
   - *Output*:
     ```text
     ==========================
     AUTHENTICATION ERROR
     ==========================
     Message: BINANCE_API_KEY is not configured or is using the default placeholder.
     ```
3. **Binance Engine Error (APIError)**:
   - *Cause*: Invalid size (smaller than minimum limit), invalid leverage, or insufficient balance.
   - *Output*:
     ```text
     -----------------------------------
     API Response
     -----------------------------------
     Success / Failure : Failure
     Error Code : -4164
     Error Message : Order's notional must be no less than 5.0 (Min Notional).
     -----------------------------------
     ```
4. **Network Timeout / Connection Drops (NetworkError)**:
   - *Cause*: Bad internet connection or incorrect DNS settings.
   - *Output*:
     ```text
     -----------------------------------
     API Response
     -----------------------------------
     Success / Failure : Failure
     Error Message : Timeout connecting to Binance Futures API at https://testnet.binancefuture.com/fapi/v1/order
     -----------------------------------
     ```

## Design Decisions & Assumptions

- **USDT-M Specific**: Symbol checks (`^[A-Z0-9]+USDT$`) ensure we match USDT-Margined futures, protecting users from accidentally ordering COIN-M pairs which have different margin profiles and specifications.
- **REST Client Selection**: Standard Python `requests` library is used. Direct signing allows the bot to run with standard python libraries, minimizing third-party risks.
- **Time In Force (GTC)**: Limit orders on Binance require a `timeInForce` parameter. We default this to `GTC` (Good Till Cancelled) to fulfill the assignment requirement simply and predictably.
- **STOP_LIMIT Mapping**: The Binance API expects conditional limit orders under the enum type `STOP`. The CLI accepts `--type STOP_LIMIT` for consumer clarity, and maps it to type `STOP` along with `stopPrice` and `price` parameters before dispatching.

## Future Improvements

1. **Precision Rounding**: Fetching ticker specifications (`GET /fapi/v1/exchangeInfo`) dynamically to round order sizes and limit prices to the exchange's minimum filters.
2. **Asynchronous Connection**: Migrating to `aiohttp` or a WebSockets stream client to allow high-frequency request processing and lower network latencies.
3. **Position Management**: Exposing endpoints for leverage configuration (`POST /fapi/v1/leverage`) and checking open positions (`GET /fapi/v1/positionRisk`).
4. **Unit Tests**: Adding a robust test suite using pytest and mocking the requests to achieve 90%+ code coverage.
